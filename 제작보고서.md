# 제작보고서 — 팝업나우(PUN)

> 팝업스토어 실시간 혼잡도 공유 웹 플랫폼
> Web Programming Term Project · 2026.06

---

## 1. 프로젝트 개요

**PUN(팝업나우)** 은 현장에 있는 유저가 직접 올린 데이터로 팝업스토어의 **혼잡도·대기팀·굿즈 재고**를 실시간으로 공유하는 풀스택 웹 서비스입니다.

| 항목 | 내용 |
|---|---|
| 서비스명 | 팝업나우 (PUN, Popup Now) |
| 한 줄 정의 | 크라우드소싱 기반 팝업스토어 실시간 혼잡도 공유 플랫폼 |
| 개발 기간 | 약 4주 (기획 → 구현 → 발표) |
| 구성 | React(Vite) + Express + MongoDB 풀스택 |
| 산출물 | 동작하는 MVP (홈 · 대시보드 · 제보 · 공지) |

---

## 2. 기획 배경

팝업스토어 방문 시 반복되는 세 가지 불편에서 출발했습니다.

1. **끝없는 대기** — 몇 팀이 밀려 있는지 가보기 전엔 알 수 없다.
2. **헛걸음** — 원하는 굿즈가 이미 품절인지 현장에서야 확인한다.
3. **흩어진 정보** — 공지·혼잡 정보가 SNS 여기저기 파편화돼 있다.

→ 현장 유저의 제보를 모아 **한곳에서 실시간으로** 확인하게 하자는 것이 핵심 아이디어입니다.

---

## 3. 주요 기능

| 기능 | 설명 |
|---|---|
| **실시간 대시보드** | 운영 중·혼잡·품절·새 공지 수를 카운트업 통계(벤토 그리드)로 표시 |
| **매장 카드** | 혼잡도 뱃지·혼잡도 게이지·대기팀 수·굿즈 재고를 3D 틸트 카드로 표현 |
| **현장 제보(FAB)** | GPS 위치 인증 후 혼잡도·대기팀·품절 여부를 직접 입력 |
| **공지 / 긴급 알림** | 운영자 공지와 긴급 알림을 매장 카드와 연동해 표시, 읽음 처리 |
| **상세 모달** | 매장 클릭 시 상세 정보 + 해당 매장 공지 아코디언 |

## 5. 시스템 구조

```
[ Client : React + Vite (5173) ]
        │  fetch /api/*  (Vite proxy)
        ▼
[ Server : Express REST API (5000) ]
        │  Mongoose
        ▼
[ Database : MongoDB ]
```

### 디렉터리 구조
```
all about project/
├── client/                 # 프론트엔드 (React + Vite)
│   ├── public/             # favicon, 아이콘
│   └── src/
│       ├── components/     # Header, StoreCard, NoticeList, DetailModal, ReportModal, Toast
│       ├── pages/          # HomePage, DashboardPage
│       ├── hooks/          # useScrollReveal, useCountUp
│       ├── App.jsx         # 라우팅 · 전역 상태 · API 호출
│       ├── main.jsx
│       └── index.css       # 디자인 시스템 전체
├── server/                 # 백엔드 (Express + MongoDB)
│   ├── models/             # Store.js, Notice.js (Mongoose 스키마)
│   ├── routes/             # stores.js, notices.js
│   ├── index.js            # 서버 진입점
│   ├── seed.js             # 초기 데이터 시드
│   └── .env.example        # 환경변수 템플릿
└── 제작보고서.md
```

### REST API
| Method | Endpoint | 설명 |
|---|---|---|
| GET | `/api/stores` | 매장 목록 조회 |
| PATCH | `/api/stores/:id` | 혼잡도·대기·품절 상태 갱신(제보) |
| GET | `/api/notices` | 공지 목록 조회 |
| PATCH | `/api/notices/:id/read` | 공지 읽음 처리 |
| POST | `/api/notices/read-all` | 모든 공지 읽음 |
| GET | `/api/health` | 서버 상태 체크 |

## 6. 디자인 시스템

최신 웹 디자인 트렌드를 반영해 단계적으로 고도화했습니다.

- **컬러** — 따뜻한 레이어드 다크 배경 + 오렌지 포인트(`#ff8a3d`), 상태색(여유/보통/혼잡 = 에메랄드/앰버/로즈)
- **글래스모피즘** — 헤더·카드·공지·모달에 프로스티드 글라스(backdrop-filter)
- **벤토 그리드** — 대시보드 통계 바를 비대칭 타일로 배치
- **타이포그래피** — 모던 한글 웹폰트 **Pretendard** 도입
- **3D 워드마크** — 히어로 우측 배경에 글로시 그라데이션 입체 "PUN" (iPhone Pro 워드마크 스타일)
- **동적 모션** — 커서 글로우, 배경 오로라 오브 패럴랙스, 스크롤 진행바, 3D 틸트 카드, 카운트업, 스크롤 리빌, 혼잡 뱃지 펄스 링, LIVE 리플, 게이지 광택 sweep, FAB 맥동 링
- **접근성** — `prefers-reduced-motion` 가드로 모션 최소화 지원

## 7. 데이터 신뢰성 설계

크라우드소싱 데이터의 한계를 숨기지 않고 **4겹의 장치**로 관리했습니다.

| 한계 | 대응 |
|---|---|
| 거짓 제보 | GPS 위치 인증 + (향후) 다수결 보정으로 이상치 제거 |
| 오래된 데이터 | 카드마다 '몇 분 전' 갱신 시각 노출 → 신선도 투명 공개 |
| 콜드 스타트 | 운영자 공지 + 유저 제보 혼합, 쓸수록 정확해지는 구조 |
| 악의적 조작 | (향후) JWT 신뢰 점수로 제보 가중치 차등 적용 |

## 8. 트러블슈팅 & 배운 점

| 문제 | 원인 | 해결 |
|---|---|---|
| 프론트–백엔드 통신 차단 | Vite(5173)·Express(5000) 포트 상이로 CORS 오류 | `cors` 미들웨어로 출처 허용, Vite proxy 설정 |
| 3D 틸트 카드 끊김 | 마우스 이동마다 리렌더로 프레임 드랍 | `requestAnimationFrame`으로 묶어 성능 최적화 |
| 스크롤 애니메이션 누락 | 동적으로 추가된 카드가 감지되지 않음 | `IntersectionObserver` 의존성 기반 재감시 |

**배운 점** — 풀스택 데이터 흐름, 비동기 상태 관리, 사용자 관점의 UI/모션 설계.

## 9. 설치 및 실행 방법

### 1) 저장소 클론
```bash
git clone <저장소 주소>
cd "all about project"
```

### 2) 백엔드
```bash
cd server
npm install
cp .env.example .env       # .env 작성 (MONGODB_URI 입력)
npm run seed               # (선택) 초기 데이터 시드
npm start                  # http://localhost:5000
```

### 3) 프론트엔드
```bash
cd client
npm install
npm run dev                # http://localhost:5173
```

> 프론트엔드의 `/api` 요청은 Vite 프록시를 통해 백엔드(5000)로 전달됩니다. 두 서버를 모두 실행해야 데이터가 표시됩니다.

## 11. AI agent 'Claude Code'의 활용
본 프로젝트를 진행하면서 AI 코딩 에이전트인 Claude Code를 개발 보조 도구로 적극 활용하였습니다. 단순히 코드를 받아 적는 방식이 아니라, 구현 방향을 직접 결정하고 에이전트에게 구체적인 요구사항을 제시하면 그에 맞춰 코드를 작성·수정하도록 하는 협업 방식으로 사용하였습니다. 특히 프론트엔드 디자인 고도화 과정에서 그 활용도가 높았는데, "최신 디자인 트렌드를 반영해 달라"는 요구에 대해 글래스모피즘, 벤토 그리드, Pretendard 폰트 적용 등 여러 방향을 제안받은 뒤 그중 원하는 방향을 선택해 단계적으로 발전시켰습니다. 또한 스크롤에 반응하는 동적 모션, 글자가 뒤에서 날아오는 등장 애니메이션, iPhone Pro 워드마크 스타일의 글로시 3D 입체 텍스트 등 추상적인 아이디어를 자연어로 설명하면 에이전트가 이를 CSS 애니메이션과 컴포넌트 코드로 구현해 주었고, 결과물을 직접 확인한 뒤 위치·색감·강도 등을 반복적으로 미세 조정하며 완성도를 높였습니다.

개발 과정에서 발생한 문제 해결에도 Claude Code를 활용하였습니다. 프론트엔드와 백엔드 서버 간 통신 오류(CORS), 3D 틸트 카드의 렌더링 성능 저하, 동적으로 추가된 요소의 스크롤 애니메이션 누락 등의 문제가 발생했을 때, 에이전트가 원인을 진단하고 requestAnimationFrame 최적화나 IntersectionObserver 재감시와 같은 해결책을 제시하여 빠르게 대응할 수 있었습니다. 이 외에도 발표용 PPT와 발표 대본 작성, 제작보고서 정리, GitHub 제출을 위한 파일 구성(민감 정보인 .env 및 node_modules 제외) 등 문서화와 형상관리 작업에도 활용하였습니다. 이러한 과정을 통해 단순 코드 생성을 넘어 기획·구현·디버깅·문서화에 이르는 전 개발 단계에서 AI 에이전트를 도구로 활용하는 경험을 쌓을 수 있었으며, 최종 의사결정과 검증은 직접 수행함으로써 결과물에 대한 이해를 유지하였습니다.
